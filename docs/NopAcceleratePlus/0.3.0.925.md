# 0.3.0.925

| **Release Date**                 | **24 April 2019** |
|----------------------------------|--------------------|
| **Plug-In Version**              | 0.3.0.925          |
| **Supported nopCommerce Version** | 4.10              |


## **Feature Highlights**

- Added **5-Star Rating** display when a product has an average rating of 5.
- Added **Setup Wizard step** for remotely hosted Solr instances.
- Implemented **Index on Entity Change** event.  
  - Enables near real-time indexing without waiting for scheduled tasks.  
  - Works only after a full indexing.  
  - Will not run during scheduled indexing.
- Performance improvement: Added setting for ignoring **Last Shopping Page**.
- Added **Scroll to Top** button on Search & Catalog pages.
- Implemented **automatic scroll-to-top after selecting a facet**.
- Implemented **Solr Down notification** (admin only).  
  - If Solr is down, fallback uses nopCommerce default search/catalog so visitors never see errors.
- Added dynamic theme compatibility fields (`ThemeFileName` & `RtlThemeFileName`) to support custom themes.
- Added **Search within Catalog** for Category, Manufacturer, Vendor & ProductTag pages.


## **Bug Fixes**

- Product box not displaying **Call for Price** — fixed.
- Plugin invoked methods even when not installed — resolved.
- Solr ping monitoring caused performance issues — replaced with faster implementation.
- Language analysis applied incorrectly to `nGram` fields, causing bad relevance — fixed.
- Spell Check with multi-term search suggested incorrect terms — corrected.
- Price range filter had fixed size — now adapts to layout.
- Tooltip on price slider not positioned correctly — fixed.
- **Mark as Pending** not working in Product Status — fixed.
- Product search inside Product Status page not working correctly — resolved.
- Wrong WidgetZone activated (Catalog instead of Search plugin) — corrected.
- “No Products Found” page should not show price slider — fixed.


## **FAQs**

### Does this release require uninstalling and reinstalling the plugin?

**Yes**

### Does this release require setting up a new core or performing re-indexing?

**Not required.**
