# 0.3.0.666

| **Release Date**                 | **15 February 2019** |
|----------------------------------|------------------------|
| **Plug-In Version**              | 0.3.0.666              |
| **Supported nopCommerce Version** | 4.10                  |


## **Feature Highlights**

- **Configure Page UI Enhancements** for better usability.
- **SEO-friendly URL structure** added.


## **Bug Fixes**

- Import/Export for Catalog Settings not working correctly — fixed.
- Catalog-wise settings not working for:  
  - Enable Infinite Scrolling  
  - Show Number of Products  
  - Enable Ajax Request  
  — resolved.
- License Key Page for Search & Catalog Plugin sometimes threw thread exceptions — fixed.
- General Settings tab moved to second position after Configure.
- Corrected spelling of **Relevancy** in sorting options on Catalog & Search pages.
- Updated “Seo” to **SEO** in General Settings tab.
- If “Enable Search” is disabled, Search Boost tab & other search-related options will now hide correctly.
- Catalog Settings now only appear when Catalog (Plus) is enabled.
- Incorrect searching for ProductAttributeValue & SpecificationAttributeValue — fixed.
- Incorrect count for “Number of Products Found” when featured products were associated with a category — resolved.
- Shorter parameter names introduced for facet URLs.
- Removed `Schedule_ManageIncrementalSolrProduct` and related button from Configure page.
- When Facet is disabled, nopCommerce native facets now load correctly and show results properly.
- Unified attribute delimiter: now using **single URL attribute delimiter** instead of three separate ones.


## **FAQs**

### Does this release require uninstalling and reinstalling the plugin?

**Yes** — because Catalog Settings were reorganized and moved, requiring a clean installation.

### Does this release require setting up a new core or performing re-indexing?

**Yes** — managed schema structure changed, so re-indexing is required.
