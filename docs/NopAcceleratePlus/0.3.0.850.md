# 0.3.0.850

| **Release Date**                 | **1 April 2019** |
|----------------------------------|------------------|
| **Plug-In Version**              | 0.3.0.850        |
| **Supported nopCommerce Version** | 4.10            |


## **Feature Highlights**

- Performance improvement: **Single combined request** for search + facet, improving response time.
- Selector configuration made **dynamic**, eliminating theme view customization in most cases.
- Added a setting to **log Solr queries** in nopCommerce system logs for debugging.
- Updated **price range slider design** with clean tooltip for min/max values.
- Improved **mobile filter UI**: slide-out menu leaves overlay space that closes when tapped.


## **Bug Fixes**

- Tours were not designed for mobile — now disabled for mobile screens.
- Added message showing indexing is in progress when user clicks "Start Indexing" while it’s already running.
- “Add to Compare” was still visible even when disabled in Catalog Settings — fixed.
- After saving settings in Catalog Settings & Catalog Facet Settings, the working tab did not reload — now fixed.
- Price range slider now respects **custom price formatting**.
- Updated “powered by” image in nopAccelerate Catalog & Search admin.
- Added logs for indexing start, stop, and failure events.
- Improved design of **License Page**.


## **FAQs**

### Does this release require uninstalling and reinstalling the plugin?

**No**

### Does this release require setting up a new core or performing re-indexing?

**No** — existing Solr Core can be used. No re-indexing required.
